0. RINGKASAN PERUBAHAN VERSI 6.0
================================================================================
1. **Top Trending** sekarang real-time dari Shopee, terbatas pada katalog kita
   (default 50 produk per hari). Daftar BERGANTI OTOMATIS setiap hari mengikuti
   pola popularitas Shopee.
2. **Produk Terlaris** juga real-time dan rotasi harian, fokus pada estimasi
   penjualan harian Shopee.
3. **Prediksi keranjang** (label "Potensi Tinggi" berbasis ATC rate) **TETAP
   DIPERTAHANKAN** sebagai akselerator estimasi penjualan.
4. **Views bulan ini** kini otomatis bertambah ketika user mengklik produk
   atau melakukan pencarian — disimpan di kolom `month_views` dan auto-reset
   tiap awal bulan.
5. **Search** sekarang mengembalikan SEMUA produk yang cocok (mis. ketik
   "hijab" → 168 hijab tampil, bukan dibatasi 50).
6. **Chatbot fallback** diperluas: setiap error Gemini (403/429/500/timeout/
   API-key leaked, dll) otomatis dialihkan ke jawaban lokal (`local_fallback_reply`)
   yang mengakses langsung 5000 produk + data SQLite. User TIDAK PERNAH lagi
   melihat pesan error dari chatbot.

3. SKEMA DATABASE (yarsimart.db, SQLite + WAL mode)
================================================================================
Database default berada di `ujicobayarsi/yarsimart.db`. Mode `PRAGMA
journal_mode=WAL` digunakan supaya banyak request bisa membaca/menulis paralel.

──────────────────────────────────────────────────────────────────────────
TABEL 1: searches
──────────────────────────────────────────────────────────────────────────
Setiap pencarian yang dilakukan user dicatat di sini.

  Kolom         Tipe                  Keterangan
  -------------------------------------------------------------------------
  id            INTEGER PRIMARY KEY   Auto-increment
  query         TEXT NOT NULL         Kata yang diketik user
  result_count  INTEGER               Jumlah produk yang cocok
  created_at    DATETIME              Waktu pencarian (ISO)

──────────────────────────────────────────────────────────────────────────
TABEL 2: product_tokens
──────────────────────────────────────────────────────────────────────────
Counter aktivitas user per produk. Inilah inti dari "real-time prediction".

  Kolom            Tipe                  Keterangan
  -------------------------------------------------------------------------
  product_id       TEXT PRIMARY KEY      P0001, P0002, ...
  tokens           INTEGER  default 0    Setiap kali muncul di hasil cari +1
  search_count     INTEGER  default 0    Sama dengan tokens (alias historis)
  views            INTEGER  default 0    Klik 'Prediksi' / kartu → +1
  cart_adds        INTEGER  default 0    Tambahkan ke keranjang → +1
  sales            INTEGER  default 0    Checkout 'buy' → +1
  month_views      INTEGER  default 0    [BARU] Aktivitas user bulan berjalan
                                          - search match  +1
                                          - klik produk   +1
                                          - cart          +2
                                          - buy           +3
  last_month_key   TEXT                  [BARU] 'YYYY-MM' marker; auto-reset
                                          month_views saat bulan ganti
  last_searched    DATETIME              Waktu interaksi terakhir

  Migrasi otomatis: kolom `month_views` & `last_month_key` di-tambahkan via
  ALTER TABLE saat startup bila DB lama tidak punya kolom tsb.

──────────────────────────────────────────────────────────────────────────
TABEL 3: keyword_counts
──────────────────────────────────────────────────────────────────────────
Token per-kata (dipecah dari query) untuk widget "Pencarian Terbanyak".

  Kolom         Tipe                  Keterangan
  -------------------------------------------------------------------------
  keyword       TEXT PRIMARY KEY      "hijab", "kemeja", ...
  count         INTEGER               Jumlah kemunculan
  last_searched DATETIME              Terakhir diketik

──────────────────────────────────────────────────────────────────────────
TABEL 4: search_product_log
──────────────────────────────────────────────────────────────────────────
Relasi many-to-many search → produk untuk audit & analitik.

  Kolom         Tipe                  Keterangan
  -------------------------------------------------------------------------
  id            INTEGER PRIMARY KEY   Auto-increment
  search_id     INTEGER → searches.id Pencarian terkait
  product_id    TEXT                  Produk yang muncul di hasil
  created_at    DATETIME              Timestamp

5. RUMUS & PERHITUNGAN PREDIKSI
================================================================================

A. RUMUS UTAMA — Prediksi Penjualan Bulan Ini (predict_product)
──────────────────────────────────────────────────────────────────────────
   month_views_user  = month_views (kolom DB, 0 jika user belum interaksi)
   cmv_total         = current_month_views (baseline) + month_views_user
   growth_rate       = cmv_total / last_month_views
   conversion_rate   = last_month_sales / last_month_views
   predicted_sales   = round( last_month_sales × growth_rate )
   predicted_daily   = round( predicted_sales / 30 )
   predicted_weekly  = round( predicted_sales / 4 )
   predicted_revenue = predicted_sales × price
   growth_pct        = (growth_rate - 1) × 100

Label tren:
   > 50%  : "Sangat Meningkat" 🟢
   > 10%  : "Meningkat" 🟢
   ±10%   : "Stabil" 🟡
   < -10% : "Menurun" 🟠
   < -30% : "Sangat Menurun" 🔴

➜ Implikasi: setiap kali user search "hijab" atau klik produk, `month_views_user`
   bertambah, `cmv_total` ikut naik, `predicted_sales` ikut naik secara real-time.

B. RUMUS BOOST PENCARIAN (predict_from_search)
──────────────────────────────────────────────────────────────────────────
   ctr        = (views    / tokens) × 100
   atc_rate   = (cart_adds/ tokens) × 100
   sales_rate = (sales    / tokens) × 100

   share_pasar = tokens / TOTAL_TOKENS_DB
   search_boost = 1.0 + share_pasar × 5
   if ctr      > 5%   → search_boost += 0.5
   if atc_rate > 2%   → search_boost += 1.0           ← prediksi keranjang
   if atc_rate > 2% and sales_rate < 1%
                      → label_prediksi = "Potensi Tinggi"

   boosted_monthly = predicted_sales × search_boost

➜ "Prediksi keranjang" yaitu deteksi produk yang banyak dimasukkan keranjang
   tapi belum banyak dibeli — diberi label "Potensi Tinggi" + boost 1.0×.
   Fitur ini DIPERTAHANKAN sesuai permintaan user agar estimasi penjualan
   lebih cepat naik.

C. SKOR TRENDING SHOPEE REAL-TIME (shopee_realtime.py)
──────────────────────────────────────────────────────────────────────────
   base_pop   = log1p(sp_qty)×60 + log1p(sp_rev)×8 + rating×5
   base_sales = sp_qty / 365                              # estimasi harian
   rotation   = 0.5 + sha256(today + salt + pid)[:12]/0xFFF...   # 0.5–1.5
   user_boost = (tokens/total_tokens)×5 + views×0.05
   live_boost = log1p(item_sold) / log1p(total_sold) bila berhasil scrape

   Trending  = (base_pop × 1.0) × rotation × (1+user_boost) × (1+live_boost)
   Bestseller= (base_pop × 0.4 + base_sales × 12) × rotation × … × …

➜ Karena `rotation` di-seed dengan TANGGAL HARI INI, daftar trending dan
   bestseller AKAN BERGANTI OTOMATIS setiap hari. Tidak perlu cron.

D. SHOPEE LIVE FETCH (best-effort)
──────────────────────────────────────────────────────────────────────────
   Endpoint    : https://shopee.co.id/api/v4/search/search_items
   Parameter   : keyword=<kategori>, by=sales, limit=20
   Auth        : tidak ada (public)
   Cache       : .shopee_cache/<keyword>_<YYYY-MM-DD>.json (per kategori per hari)
   Fallback    : bila gagal (anti-bot, jaringan, dll) → live_boost = 0,
                 jadi sistem tetap pakai daily rotation + baseline + user boost.

➜ Default `use_shopee_live=true`. Bisa dimatikan via query
   `/api/trending?use_live=0` jika dev mode tidak butuh akses internet.

E. ESTIMASI PENJUALAN HARIAN SHOPEE
──────────────────────────────────────────────────────────────────────────
   daily_estimated_sales(p, meta) = base_sales × rotation × (1 + user_boost)
                                  ≈ (sp_qty / 365) × faktor_hari

6. DAFTAR API ENDPOINT (Flask)
================================================================================
GET  /                          Render index.html (SPA)

POST /api/search                Body: {"query": "..."}
                                 ▸ Cari produk (return SEMUA yang match)
                                 ▸ Update searches, product_tokens, keyword_counts
                                 ▸ Tambah month_views +1 untuk tiap match
                                 Response: {query, count, results[], search_id, message}

GET  /api/trending?limit=50&use_live=1
                                 ▸ Top N trending real-time Shopee dari katalog
                                 ▸ Skor = baseline × daily_rotation × (1+user_boost) × (1+live)
                                 ▸ Daftar berganti tiap hari otomatis
                                 Response: {trending[], day, source, count}

GET  /api/bestsellers?limit=10&use_live=1
                                 ▸ Top N produk terlaris hari ini (Shopee)
                                 ▸ Termasuk shopee_daily_est & label_prediksi
                                 Response: {bestsellers[], day, source, count}

GET  /api/keywords               Top 15 kata kunci dari keyword_counts
GET  /api/stats                  Statistik global (total_searches, total_tokens, dll)
GET  /api/predict/<pid>          Prediksi lengkap satu produk (cmv augmented user)
GET  /api/predict_all?cat=&store=&page=1&per_page=100
                                 Prediksi paginated semua produk
GET  /api/category_summary       Aggregasi per kategori (untuk Tabel 1–4 Analitik)
GET  /api/store_summary          Aggregasi per toko

POST /api/action                 Body: {"pid":..., "action":"view"|"cart"|"buy"}
                                 ▸ Increment counter (& month_views: view+1, cart+2, buy+3)

POST /api/reset                  Reset semua tabel (kecuali katalog)

POST /api/chat                   Body: {"message":"...", "history":[...]}
                                 ▸ Coba 3 model Gemini berurutan
                                 ▸ Jika SEMUA gagal → fallback lokal
                                 ▸ Status: "ok" | "local"
                                 ▸ User tidak pernah dapat 5xx error

7. ALUR FRONTEND (static/js/app.js)
================================================================================
Halaman   : SPA dengan 4 tab (Beranda / Prediksi / Analitik / Produk)
Init      : window.onload → fillSelect, renderKPI, renderAllProducts,
            refreshDashboard (setiap 15 detik)

Fungsi penting:
  performSearch()      Kirim query ke /api/search, render SEMUA hasil di grid
  showSuggestions()    Auto-complete on-the-fly dari PRODUCTS object
  refreshDashboard()   Pull /api/stats, /api/keywords, /api/trending, /api/bestsellers
  openPrediction(pid)  Pindah ke tab Prediksi & render data live
  renderPrediction()   POST 'view' → GET /api/predict/<pid> → render kartu
                       (gunakan cmv dari server, bukan data.js statis)
  renderTrending()     Render top 10 trending dengan badge LIVE Shopee
  renderBestsellers()  Render bestseller dengan label_prediksi & shopee_daily_est
  renderAnalytics()    Render 4 tabel + 4 chart Chart.js
  sendMsg()            Chatbot — ke /api/chat, fallback ke local jika reply error

Catatan: variable `PRODUCTS` di data.js memakai short keys (n,c,s,pr,…).
Kolom prediksi yang dipakai dari server menang: lmv, lms, cmv, cmv_user.

8. CHATBOT — STRATEGI 2-LAPIS (TIDAK PERNAH GAGAL)
================================================================================
Lapis 1: GOOGLE GEMINI (online)
  Header   : x-goog-api-key + Content-Type: application/json
  URL      : https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent
  Models   : gemini-2.0-flash → gemini-1.5-flash → gemini-flash-latest
  Timeout  : 30 detik per model
  Prompt   : system_prompt mengandung statistik DB + Top kategori + Top 10 produk
  History  : 8 pesan terakhir di-include sebagai context

Lapis 2: LOCAL FALLBACK (offline, selalu jalan)
  Trigger  : *setiap* error apa pun (HTTPError 403/429/500, timeout, JSON error,
             API key dilaporkan leaked, dll). Kode berlapis 2x:
               1. inner loop: kalau error, lanjut ke model berikutnya
               2. outer try/except: kalau ada exception tak terduga, fallback dipanggil
  Fungsi   : local_fallback_reply(user_message, stats)
  Logic    : keyword matching pada user_message →
               "terlaris/bestseller/laku" → top 5 by sp_rev
               "mahal/premium/termahal"   → top 5 by price (desc)
               "murah/terjangkau/hemat"   → top 5 by price (asc)
               "pendapatan/revenue/omzet" → total qty/revenue 5000 produk
               "trending/populer"          → ringkasan stats
               "prediksi/forecast/proyeksi"→ panggil predict_product untuk match
               "toko/store/seller"         → cari nama toko spesifik
               <kategori>                  → top 5 produk kategori tsb
               <produk>                    → detail satu produk (predict)
               default                     → daftar bantuan

Output: HTML ber-bold + footnote "Mode data cadangan aktif"
